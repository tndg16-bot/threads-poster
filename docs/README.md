# Threads投稿管理 - ドキュメント

## CSV を Google Sheets にインポートする方法

### 手順

1. Google Sheets で「Threads投稿管理」スプレッドシートを開く
2. メニューから **ファイル > インポート** を選択
3. **アップロード** タブで `april-2026-schedule.csv` を選択
4. インポート設定:
   - **インポート先**: `現在のスプレッドシートに新しいシートを挿入する` または `特定のセル範囲に挿入する`（既存シートに追記する場合）
   - **区切り文字の種類**: `カンマ`
   - **テキストを数値、日付、数式に変換する**: `はい`
5. **データをインポート** をクリック

### 既存シートへの追記

既存の「4月」シートにデータを追記したい場合は、以下の手順が確実:

1. CSV をいったん新規シートとしてインポート
2. ヘッダー行を除いたデータ行をコピー
3. 既存シートの最終行の下にペースト
4. インポート用の一時シートを削除

## スプレッドシートのカラムスキーマ

| カラム名 | 型 | 説明 | 例 |
|---------|------|------|------|
| 日付 | DATE | 投稿予定日（YYYY-MM-DD形式） | `2026-04-01` |
| 投稿時間 | TIME | 投稿時刻（HH:MM形式） | `07:00` / `20:00` |
| 投稿文 | TEXT | Threads投稿本文（500文字以内） | 日本語テキスト |
| カテゴリ | TEXT | ブログカテゴリ | `AI活用`, `マインドセット`, `生産性` 等 |
| トピックタグ | TEXT | 投稿のタグ（50文字以内、ドットやアンパサンド不可） | `AIネイティブな働き方` |
| 記事URL | URL | 紐づくブログ記事のURL | `https://takahiro-motoyama.vercel.app/blog/...` |
| 画像URL | URL | OGP画像URL（空欄の場合は記事URLから自動取得） | 空欄 or URL |
| ステータス | TEXT | 投稿の状態 | `下書き` → `予約済み` → `投稿済み` → `失敗` |
| 投稿ID | TEXT | Threads投稿後に自動記録されるID | 投稿後に自動入力 |
| メモ | TEXT | 管理用メモ（投稿パターン名など） | `質問型`, `リスト型`, `体験談型` 等 |

### ステータスの遷移

```
下書き → 予約済み → 投稿済み
                  ↘ 失敗（リトライ可）
```

- **下書き**: 初期状態。内容のレビュー・編集が可能
- **予約済み**: n8n ワークフローによる自動投稿キューに入った状態
- **投稿済み**: Threads への投稿が完了。投稿IDが自動記録される
- **失敗**: 投稿エラー。メモ欄にエラー内容が記録される

## Web UI との連携

### 概要

`threads-poster/web/` にある Web UI は、スプレッドシートのデータを可視化・操作するためのインターフェース。

### 使い方

1. スプレッドシートで投稿スケジュールを管理
2. Web UI で投稿プレビュー・ステータス確認
3. 即時投稿やスケジュール変更は Web UI から操作可能

## n8n ワークフローとの連携

### 概要

`threads-poster/n8n/` にあるワークフローが、スプレッドシートのデータを定期的にチェックし、予約時刻に自動投稿を実行する。

### 処理フロー

1. **スケジュールトリガー**: n8n が定期的（5分間隔）にスプレッドシートをチェック
2. **対象行の抽出**: `ステータス = 予約済み` かつ `投稿時間` が現在時刻を過ぎている行を取得
3. **OGP画像取得**: `画像URL` が空の場合、`記事URL` から OGP 画像を自動取得
4. **Threads API 投稿**: Threads Publishing API 経由で投稿を実行
5. **結果の書き戻し**: 成功時は `ステータス = 投稿済み` + `投稿ID` を記録、失敗時は `ステータス = 失敗` + エラー内容をメモに記録

### 運用手順

1. CSV をインポートして投稿スケジュールを作成
2. 投稿内容をレビュー・編集
3. 投稿する行の `ステータス` を `予約済み` に変更
4. n8n ワークフローが自動で投稿を実行
5. 結果をスプレッドシートで確認

## 投稿パターン一覧

曜日ごとに異なる投稿パターンを使い分け、フィードの多様性を確保する。

| 曜日 | パターン | 特徴 |
|------|---------|------|
| 月曜 | 共感型 | 読者の悩みに寄り添い、共感から入る |
| 火曜 | Tips型 / リスト型 | 箇条書きで実用的な情報を提供 |
| 水曜 | 逆説型 / 数字型 | 常識を覆す切り口や具体的な数字で惹きつける |
| 木曜 | 質問型 | 読者に問いかけ、考えさせる |
| 金曜 | 体験談型 / Before After型 | 実体験ベースのストーリーで信頼感を醸成 |
